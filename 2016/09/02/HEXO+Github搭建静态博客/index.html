HEXO+Github搭建静态博客

Sep 2, 2016 | hexo | Hits

　　踩过千坑万坑，终于搭建好了hexo，暂时效果就是这样(我知道特别丑。。)　　域名直接就在DNSPOD上买的，买完刚好直接在这上面进行DNS设置什么的，很方便，支持支付宝和微信支付，懒人必备。。。

我的hexo是3.2.2版本（在git中输入hexo version 查看自己的hexo当前版本），hexo不同的版本，很多配置都不一样，3.X版本的改动不是很大，各位搭建自己网站的时候，一定要看新版本的教程。。不然老版本的看起来经常会出现云里雾里的状况。。我的电脑是Windows7 64位系统。其他的常见系统按照教程来一般也是没有问题的。

正文：

hexo是一款基于Node.js的静态博客框架，我相信，只要你想搭建，肯定有成千上万的教程，所以，我在这里不再赘述，仅仅是推荐几个我在搭建过程中对我很有帮助的教程：

在跟随教程的搭建过程中，你肯定会遇到各式各样的问题，“明明博主搭建过程好好的，为什么我找他的方法就是不行呢？ WTF…”

　　出现问题之后，首先要做的，就是仔细对照教程中的步骤与你的操作，有没有漏掉什么，如果发现是自己的过程有误，改过来就好了；

　　如果还是解决不了的话，百度是个好工具，如果有vpn比如影梭或者google镜像之类的神器，那就更应该多用。 搭建博客的人千千万，你的问题肯定不是什么新问题，多看看别人的博客，或者csdn，或者segmentfault等大牛常出没的地方，没准就在哪发现你的问题所在了。如果英文过关的话，可以去程序问题天堂stackoverflow和github去搜一搜问一问，肯定会得到你想要的；

　　如果，如果上面两步都没有解决你的问题的话，不要气馁，真正的勇士敢于面对惨淡的人生，删掉当前的目录，换个教程，再来一遍就是了。

我在建站的过程中，代码上传github的时候遇到一个非常蛋疼菊紧的问题，配置Github过程中，
deploy:

type: git

repo: https://github.com/AyoCross/AyoCross.github.io.git

branch: master

配置完成之后，向git提交：

hexo deploy

死活提交不上，出现的异常看不懂。一遍遍找方法试，但总是不行。一直提示

fatal: could not read Username for ‘https://github.com‘: Invalid argument

delete mode 100644 placeholder
warning: LF will be replaced by CRLF in js/script.js.
The file will have its original line endings in your working directory.

error: failed to execute prompt script (exit code 1)
fatal: could not read Username for 'https://github.com': Invalid argument
FATAL Something's wrong. Maybe you can find the solution here: http://hexo.io/docs/troubleshooting.html
Error: bash: /dev/tty: No such device or address
